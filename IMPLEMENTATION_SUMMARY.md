**Файл:** `core/src/execute.c`, `core/include/execute.h`

**Изменения:**

  - `MOV Rd, #imm8` — загрузка константы

### 3. Go-оркестратор (новый компонент)

**Директория:** `orchestrator/`

**Структура:**
```
orchestrator/
├── main.go              # Точка входа, CLI
├── go.mod               # Зависимости Go
├── config.yaml          # Конфигурация
├── types/
│   └── types.go         # Task, TaskResult структуры
├── keydb/
│   └── client.go        # Клиент для Redis/KeyDB
└── simulator/
    └── runner.go        # Запуск симулятора, парсинг вывода
```

**Функционал:**
- ✅ Два режима работы:
  - `queue` — очередь заданий (KeyDB/Redis)
  - `file` — файлы (для тестов)
- ✅ Запуск C-симулятора как процесса
- ✅ Обработка таймаутов
- ✅ Парсинг вывода симулятора
- ✅ Отправка результатов в очередь

**Пример использования:**
```bash
# Режим файла
./orchestrator --mode file --file task.json --output result.json

# Режим очереди
./orchestrator --mode queue --redis localhost:6379
```

## 🔄 Как это работает вместе

### Полный цикл выполнения задачи

```
1. Студент
   ↓ (загружает .bin файл)
   
2. Сервер ITMO.clab
   ↓ (кодирует в base64, создаёт JSON)
   
3. KeyDB/Redis Queue: "simulator:tasks"
   ↓ (LPUSH)
   
4. Go-оркестратор
   ↓ (BLPOP, получает задачу)
   
5. Создаёт временный файл с .bin
   ↓
   
6. Запускает C-симулятор
   ↓ (exec.Command, таймаут)
   
7. C-симулятор выполняет код
   ↓ (stdout: статистика, логи)
   
8. Go-оркестратор парсит вывод
   ↓ (regex: [STATS], [GPIO], [UART])
   
9. Отправляет результат в KeyDB
   ↓ (RPUSH: "simulator:results")
   
10. Сервер забирает результат
    ↓ (RPOP)
    
11. Студент получает оценку
```

## 🚀 Как запускать

### Быстрая проверка (Linux/WSL)

```bash
./quick_test.sh
```

### Быстрая проверка (Windows PowerShell)

```powershell
.\quick_test.ps1
```

### Ручная сборка и запуск

```bash
# Сборка симулятора
mkdir build && cd build
cmake ..
make

# Запуск демо
./stm32_sim --demo

# Создание тестового бинарника
python3 ../create_test_bin.py demo

# Запуск тестового бинарника
./stm32_sim ../demo.bin

# Запуск тестов
./test_gpio
./test_simulator

# Сборка оркестратора
cd ../orchestrator
go mod download
go build -o orchestrator .

# Запуск оркестратора (file mode)
./orchestrator --mode file --file ../test_task.json --output result.json --simulator ../build/stm32_sim
```

## 📚 Полезные команды

```bash
# Показать справку симулятора
./stm32_sim --help

# Запустить с ограничением инструкций
./stm32_sim program.bin --max-steps 1000

# Проверить версию Go
go version

# Запустить Redis (Docker)
docker run -d -p 6379:6379 redis:latest

# Отправить задачу в Redis
redis-cli LPUSH simulator:tasks '{"task_id":"test","binary":"BQUDGII=","timeout_sec":5}'

# Получить результат из Redis
redis-cli RPOP simulator:results
```

## 🎓 Образовательная ценность

Этот проект демонстрирует:

1. **Архитектуру Cortex-M3** — регистры, память, инструкции
2. **Системное программирование на C** — работа с памятью, битовые операции
3. **Микросервисы на Go** — конкурентность, работа с очередями
4. **Контейнеризацию** — Docker, multi-stage build
5. **Интеграцию систем** — C + Go + Redis + Web UI
