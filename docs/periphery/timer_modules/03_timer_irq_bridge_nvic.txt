Модуль: 03_timer_irq_bridge_nvic

Назначение
- Преобразует состояние TIM6 (UIF/UIE) в pending-событие NVIC.

Область ответственности
- Выставление pending для TIM6 IRQ при условии: SR.UIIF=1 и DIER.UIE=1.
- Очистка pending при принятии IRQ CPU/NVIC-политикой.
- Исключение прямого управления CPU-контекстом.

Интерфейс (ориентир на текущий проект)
- Взаимодействие с nvic.c/nvic_extended.c через существующие API pending/enable.
- tim6_irq_update_line(TIM6_State*, NVIC_State*): пересчет IRQ-состояния (рекомендуемый слой интеграции).

Требования
- SR.UIF отражает факт update независимо от DIER.UIE.
- DIER.UIE управляет только генерацией IRQ-запроса.
- Если pending уже установлен, повторные события не теряются минимум как факт «есть необслуженное событие».
- Таймерный модуль не выбирает приоритет и не выполняет arbitration IRQ.

Критерии приемки
- UIF=1 и UIE=0: pending не ставится.
- UIF=1 и UIE=1: pending ставится.
- После обслуживания и очистки UIF pending снимается по принятой NVIC-политике.
