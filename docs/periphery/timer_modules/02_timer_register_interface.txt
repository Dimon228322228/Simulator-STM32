Модуль: 02_timer_register_interface

Назначение
- Предоставляет MMIO-интерфейс доступа к регистрам таймера из CPU/памяти.
- Валидирует чтение/запись и синхронизирует регистры с core-моделью.

Область ответственности
- Карта регистров TIM6: CR1, DIER, SR, PSC, ARR, CNT (минимум).
- Логика read/write для каждого регистра.
- Правильная очистка/установка битов (например, UIF).

Необходимые функции
- tim6_read32(addr): чтение 32-битного значения регистра.
- tim6_write32(addr, value): запись 32-битного значения регистра.
- tim6_apply_reset_values(): установка reset-значений регистров.
- tim6_after_write_hooks(reg): пост-обработка записей, влияющих на core (например CEN/UIE).

Требования
- Запись в неизвестные адреса TIM6 должна обрабатываться безопасно (ошибка).
- UIF очищается программой только по согласованному правилу (например write-0-to-clear).
- Изменение PSC/ARR во время работы должно иметь определенное поведение (применение на update event).
- Регистровая логика не должна напрямую выполнять IRQ-вход CPU, только выставлять состояние для IRQ-моста.

Критерии приемки
- Чтение/запись всех поддержанных регистров проходят unit-тесты.
- Очистка UIF работает строго по специфицированному правилу.
- Невалидные доступы не ломают симуляцию.
