Модуль: 06_waveform_tracing_gtkwave

Назначение
- Дает наблюдаемость таймера и IRQ-цепочки в GTKWave (VCD/FST).

Область ответственности
- Экспорт сигналов TIM6/CPU/NVIC на каждом instruction-step.
- Проверка временных соотношений и корректного ISR-flow.

Интерфейс интеграции
- Инициализация трассы в старте симуляции.
- Запись значений после каждого шага execute/timer update.
- Корректное закрытие трассы по завершении прогона.

Обязательные сигналы
- instruction_tick (или эквивалент шага CPU)
- tim6_cr1_cen
- tim6_psc, tim6_arr, tim6_cnt
- tim6_dier_uie, tim6_sr_uif
- tim6_irq_pending (или линия NVIC pending для TIM6)
- cpu_in_isr
- cpu_pc (желательно)

Требования
- Изменения состояния не должны теряться между шагами.
- Временная ось интерпретируется как instruction-step.
- В отчете фиксируется: T_update_ticks = (PSC + 1) * (ARR + 1).

Критерии приемки
- На графике видны: счет CNT, установка UIF, pending IRQ, вход в ISR, очистка UIF.
- Для разных PSC/ARR период на графике совпадает с формулой.
