Модуль: 04_cpu_interrupt_dispatcher_no_nesting

Назначение
- Реализует вход/выход из ISR без вложенных прерываний.

Область ответственности
- Проверка pending IRQ между инструкциями.
- Вход в ISR только если CPU не в обработчике (in_isr=false).
- Сохранение/восстановление контекста CPU.

Интерфейс интеграции
- Использует существующие структуры CPU/NVIC (cpu_state.h, nvic.h, nvic_extended.h).
- Точка вызова: цикл execute (fetch/execute), где после завершения инструкции проверяется pending.

Требования
- Текущая инструкция завершается до входа в ISR.
- Пока in_isr=true, новые IRQ только накапливаются в pending.
- После return из ISR следующий pending может быть принят на следующем instruction-step.
- Поведение одинаково для периодических и отложенных (one-shot) событий.

Критерии приемки
- Два IRQ подряд обрабатываются последовательными ISR-входами без вложения.
- Флаг in_isr корректно держится true весь ISR.
- После возврата выполнение основной программы продолжается с корректного PC.
